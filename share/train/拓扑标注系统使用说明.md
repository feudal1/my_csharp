# 零件拓扑标注系统使用说明

## 概述

本系统用于标注和管理 SolidWorks 零件的拓扑结构特征。通过 Weisfeiler-Lehman (WL) 图核算法计算零件的拓扑图，并允许用户为每个零件添加自定义标签，便于后续的相似性检索和分类。

## 核心功能

### 1. 拓扑图计算
- 从 SolidWorks 零件提取面邻接图
- 使用 WL 算法迭代计算节点标签
- 生成零件的拓扑特征向量

### 2. 数据持久化
- SQLite 数据库存储零件信息和标注
- 支持多次迭代结果的完整保存
- 自动管理标注版本和历史

### 3. 交互式标注
- 命令行交互式标注界面
- 支持多种标注类别
- 可设置置信度和备注说明

### 4. 查询与统计
- 按标注类别搜索零件
- 查看所有已标注零件
- 数据库统计信息

## 命令列表

### label - 标注当前零件
**功能**: 打开交互式标注界面，计算当前零件的拓扑图并添加标注

**用法**:
```bash
label
```

**操作流程**:
1. 在 SolidWorks 中打开零件
2. 运行 `label` 命令
3. 系统自动计算 WL 拓扑特征
4. 根据提示输入标注类别和值
5. 可选择添加置信度和备注

**示例**:
```
=== 零件拓扑标注 ===

零件 [底板.sldprt] 构建完成：24 个面

=== 执行 WL 迭代 (1 次) ===
  迭代 0: 5 种标签
  迭代 1: 12 种标签

=== 存储到数据库 ===
✓ 零件 '底板.sldprt' 已存入数据库 (2 次迭代)

=== 添加标注 ===
请输入标注类别（直接回车跳过）:
标注类别 > 结构类型
标注值 [结构类型] > 框架
置信度 [0-1, 默认 1.0] > 0.9
备注说明（可选） > 主要支撑结构

标注类别 > 材料
标注值 [材料] > Q235A
置信度 [0-1, 默认 1.0] > 
备注说明（可选） > 

标注类别 > 
✓ 标注完成
```

### label_quick - 快速标注
**功能**: 仅计算并存储拓扑特征，不立即添加标注

### label_search - 搜索零件
**功能**: 按标注类别和可选的值搜索零件

**用法**:
```bash
label_search [类别名称] [可选的值]
```

**参数**:
- `类别名称`: 标注的类别（如"结构类型"、"材料"等）
- `可选的值`: 具体的标注值（如"框架"、"Q235A"等）

**示例**:
```bash
# 搜索所有标注了"结构类型"的零件
label_search 结构类型

# 搜索结构类型为"框架"的零件
label_search 结构类型 框架

# 搜索材料为"Q235A"的零件
label_search 材料 Q235A
```

### label_batch - 批量标注文件夹
**功能**: 批量处理文件夹中的所有 SolidWorks 零件

**用法**:
```bash
label_batch [文件夹路径]
```

**参数**:
- `文件夹路径`: 包含 .sldprt 或 .sldasm 文件的文件夹

**示例**:
```bash
label_batch E:\projects\parts
```

**说明**:
- 自动扫描文件夹中的所有零件
- 计算并存储每个零件的 WL 特征
- 不添加用户标注（可后续用 `label` 命令补充）

**参数**:

## 标注类别建议

以下是推荐的标注类别，可根据实际需求调整：

### 结构特征类
- **结构类型**: 框架、板材、轴类、壳体、支架等
- **主要形状**: 长方体、圆柱体、不规则等
- **复杂度**: 简单、中等、复杂

### 材料工艺类
- **材料**: Q235A、45 钢、铝合金、不锈钢等
- **表面处理**: 镀锌、喷漆、氧化、发黑等
- **加工工艺**: 铸造、锻造、焊接、机加工等

### 功能用途类
- **用途**: 支撑、连接、传动、密封等
- **工作条件**: 常温、高温、高压、腐蚀等
- **重要程度**: 关键件、重要件、一般件

### 设计属性类
- **设计者**: 设计师姓名
- **项目**: 所属项目名称
- **版本**: 设计版本号

## 数据库文件

### 位置
数据库文件默认保存在程序运行目录：`topology_labels.db`

### 表结构
**parts 表** - 零件基本信息
- `id`: 主键
- `part_name`: 零件名称
- `file_path`: 文件路径
- `thickness`: 厚度
- `created_at`: 创建时间
- `updated_at`: 更新时间

**wl_results 表** - WL 迭代结果
- `id`: 主键
- `part_id`: 零件 ID（外键）
- `iteration`: 迭代次数
- `label_frequencies`: 标签频率（JSON 格式）

**user_labels 表** - 用户标注
- `id`: 主键
- `part_id`: 零件 ID（外键）
- `label_category`: 标注类别
- `label_value`: 标注值
- `confidence`: 置信度（0-1）
- `notes`: 备注说明
- `created_at`: 创建时间

## 最佳实践

### 1. 标注流程
```
打开零件 → label_quick（快速存储）→ view_parts（查看）→ 
label（补充标注）→ label_stats（检查统计）
```

### 2. 批量处理
```
label_batch [文件夹] → view_parts（检查）→

```

```

```

## 常见问题

### Q: 标注可以修改吗？
A: 可以。系统支持为同一类别添加多个标注，每次标注都会保留历史记录。查询时默认返回最新的标注。

### Q: 如何删除标注？
A: 目前不支持删除操作。如需删除，可使用 SQLite 数据库管理工具直接操作数据库文件。

### Q: 标注的置信度有什么作用？
A: 置信度用于标识标注的可靠程度，可在后续开发中用于加权计算或筛选高质量标注。

### Q: 能否自定义标注类别？
A: 可以。标注时在"标注类别"提示中输入任意类别名称即可，系统会自动识别并存储。

### Q: WL 迭代次数应该设为多少？
A: 通常 1-2 次迭代即可捕捉足够的拓扑信息。迭代次数越多，计算时间越长，但能捕获更复杂的结构特征。

## 技术细节

### WL 图核算法
Weisfeiler-Lehman 算法通过迭代更新图中节点的标签来捕获图的拓扑结构：
1. 初始标签：每个面的几何类型（平面、圆柱面等）
2. 迭代更新：结合邻居标签生成新标签
3. 结果表示：每次迭代后的标签频率分布

### 相似度计算
基于 WL 迭代的标签频率向量，使用余弦相似度计算零件间的拓扑相似度：
- 余弦相似度：cos(θ) = (A·B) / (||A|| × ||B||)
- 值域：[0, 1]，越大表示越相似

### 性能优化
- 使用 SQLite 索引加速查询
- JSON 格式存储频率数据，节省空间
- 支持增量更新，避免重复计算
