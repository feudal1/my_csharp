## 🎯 任务定义

这是一个**图上的链接预测 (Link Prediction)** 任务：

- **输入**: 零件的面邻接图 + 几何特征
- **输出**: 预测哪两个面之间需要标注尺寸
- **本质**: 在图中预测"应该存在但当前不存在"的边（标注关系）

### **第三步：模型架构**

我推荐三种方案，从简单到复杂：

#### **方案 A: 二元分类器（最简单，适合入门）**

把问题看作：**给定两个面，预测它们之间是否需要标注**

```python
import torch
import torch.nn as nn

class DimensionPredictor(nn.Module):
    def __init__(self, face_feature_dim, pair_feature_dim):
        super().__init__()
        
        # 面特征编码器
        self.face_encoder = nn.Sequential(
            nn.Linear(face_feature_dim, 64),
            nn.ReLU(),
            nn.Linear(64, 64)
        )
        
        # 组合特征预测
        self.classifier = nn.Sequential(
            nn.Linear(64 * 2 + pair_feature_dim, 128),  # 两个面 + pair 特征
            nn.ReLU(),
            nn.Dropout(0.3),
            nn.Linear(128, 64),
            nn.ReLU(),
            nn.Linear(64, 1),  # 输出概率
            nn.Sigmoid()
        )
    
    def forward(self, face1_feat, face2_feat, pair_feat):
        # 编码单个面特征
        emb1 = self.face_encoder(face1_feat)
        emb2 = self.face_encoder(face2_feat)
        
        # 拼接特征
        combined = torch.cat([emb1, emb2, pair_feat], dim=-1)
        
        # 预测
        return self.classifier(combined)

# 使用示例
model = DimensionPredictor(face_feature_dim=15, pair_feature_dim=10)
prob = model(face1_features, face2_features, pair_features)
# prob > 0.5 表示需要标注
```

**优点**: 
- 简单易实现
- 可以直接用你现有的特征

**缺点**: 
- 需要枚举所有面 pair 进行预测（O(n²) 复杂度）
- 忽略了图的全局结构

#### **方案 B: 图神经网络 + 链接预测（推荐）**

使用 GNN 学习面的嵌入，然后预测边：

```python
import torch
from torch_geometric.nn import GCNConv, GraphSAGEConv
from torch_geometric.utils import negative_sampling

class GNNDimensionPredictor(torch.nn.Module):
    def __init__(self, node_feature_dim, edge_feature_dim):
        super().__init__()
        
        # 图卷积层
        self.conv1 = GraphSAGEConv(node_feature_dim, 128)
        self.conv2 = GraphSAGEConv(128, 128)
        
        # 边分类器（预测两个节点之间是否有标注）
        self.edge_classifier = nn.Sequential(
            nn.Linear(128 * 2 + edge_feature_dim, 256),
            nn.ReLU(),
            nn.Dropout(0.3),
            nn.Linear(256, 128),
            nn.ReLU(),
            nn.Linear(128, 1)
        )
    
    def decode(self, x, edge_index, edge_feat):
        """
        x: 节点嵌入
        edge_index: 边的连接关系
        edge_feat: 边特征
        """
        # 获取边两端的节点嵌入
        src, dst = edge_index
        
        # 拼接源节点、目标节点和边特征
        edge_embed = torch.cat([x[src], x[dst], edge_feat], dim=-1)
        
        return self.edge_classifier(edge_embed).sigmoid()
    
    def forward(self, data):
        # 图编码
        x, edge_index, edge_feat = data.x, data.edge_index, data.edge_attr
        
        x = self.conv1(x, edge_index).relu()
        x = self.conv2(x, edge_index).relu()
        
        # 解码（预测边）
        return self.decode(x, edge_index, edge_feat)

# 训练循环
model = GNNDimensionPredictor(node_feature_dim=15, edge_feature_dim=10)
optimizer = torch.optim.Adam(model.parameters(), lr=0.001)

for epoch in range(100):
    model.train()
    optimizer.zero_grad()
    
    # 正样本（已有的标注）
    pos_edge_index = data.positive_edges  # 需要标注的面 pair
    pos_edge_score = model(data.x, pos_edge_index, data.positive_edge_feat)
    
    # 负样本（随机采样的面 pair）
    neg_edge_index = negative_sampling(pos_edge_index, num_nodes=data.num_nodes)
    neg_edge_score = model(data.x, neg_edge_index, data.negative_edge_feat)
    
    # 二元交叉熵损失
    loss = nn.BCELoss()(pos_edge_score, torch.ones_like(pos_edge_score)) + \
           nn.BCELoss()(neg_edge_score, torch.zeros_like(neg_edge_score))
    
    loss.backward()
    optimizer.step()
```

**优点**:
- 利用了图的拓扑结构
- 可以批量预测所有可能的面 pair
- 效果通常比纯分类器好

#### **方案 C: Transformer + 对比学习（最先进）**

如果你的数据量很大（>1000 个零件），可以考虑：

```python
import torch
from transformers import GraphormerModel

class GraphFormerDimensionPredictor(nn.Module):
    def __init__(self):
        super().__init__()
        # 使用预训练的图 Transformer
        self.backbone = GraphormerModel.from_pretrained('microsoft/graphormer-base')
        self.predictor = nn.Linear(768, 1)
    
    def forward(self, graph_data):
        outputs = self.backbone(
            input_ids=graph_data.node_features,
            attention_mask=graph_data.adjacency_matrix
        )
        
        # 池化为图级表示
        graph_embed = outputs.last_hidden_state.mean(dim=1)
        
        return self.predictor(graph_embed).sigmoid()
```

```python
def predict_dimensions_for_part(part_doc, trained_model):
    """
    对新零件预测需要添加的标注
    """
    # 1. 提取所有面
    faces = get_all_faces(part_doc)
    
    # 2. 构建完全图（所有可能的面 pair）
    face_pairs = list(combinations(faces, 2))
    
    # 3. 提取特征
    features = extract_pair_features(face_pairs)
    
    # 4. 预测
    probabilities = trained_model.predict(features)
    
    # 5. 筛选高置信度的预测
    threshold = 0.7  # 可调整
    predicted_dims = [
        (face1, face2, prob) 
        for (face1, face2), prob in zip(face_pairs, probabilities)
        if prob > threshold
    ]
    
    # 6. 自动创建标注（可选）
    for face1, face2, confidence in predicted_dims:
        create_dimension_in_drawing(part_doc, face1, face2)
    
    return predicted_dims
```

## ⚠️ 关键挑战与解决方案

### **挑战 1: 标注标准不一致**
不同工程师可能标注不同的面

**解决**: 
- 只收集"公认"的标注（多人标注的一致性区域）
- 或者给每个样本加"置信度"权重

### **挑战 2: 上下文依赖**
某些标注取决于视图方向或加工工艺

**解决**:
- 加入视图方向作为特征
- 按加工类型分组训练（车削件/铣削件/钣金件）

### **挑战 3: 长尾分布**
大部分标注是线性尺寸，少数是角度/半径等

**解决**:
- 分层采样（保证每种类型都有足够样本）
- 多任务学习（同时预测标注类型和位置）
