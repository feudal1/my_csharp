# 零件处理状态任务窗格使用说明

## 功能概述

这个任务窗格用于记录和跟踪SolidWorks中零件的处理状态，方便批量处理零件时监控进度。

## 主要文件

1. **PartStatusForm.cs** - 任务窗格的窗体类，包含状态显示界面
2. **addin.cs** - 添加了任务窗格的初始化和清理逻辑
3. **function.cs** - 添加了两个新命令来控制任务窗格

## 新增命令

### 1. 显示零件状态窗格 (ID: 1017)
- **命令代号**: `show_part_status`
- **功能**: 显示或激活零件处理状态任务窗格
- **使用**: 在SolidWorks工具栏中点击此按钮，任务窗格会显示在右侧

### 2. 记录零件处理状态 (ID: 1018)
- **命令代号**: `record_part_status`
- **功能**: 将当前打开的文档状态记录到任务窗格中
- **使用**: 
  1. 打开任意SolidWorks文档（零件/装配体/工程图）
  2. 点击此命令
  3. 任务窗格会自动记录零件名称、状态、时间和文档类型

## 任务窗格功能

### 界面组成
- **状态表格**: 显示所有记录的零件状态
  - 零件名称
  - 处理状态
  - 处理时间
  - 备注信息
  
- **操作按钮**:
  - **刷新**: 重新显示所有状态记录
  - **清空**: 清除所有记录（需确认）

### 状态颜色标识
- 🟢 **浅绿色**: 成功/完成状态
- 🔴 **浅红色**: 失败/错误状态  
- 🟡 **浅黄色**: 处理中状态

## 编程接口

在其他命令中可以使用以下API来记录状态：

```csharp
// 获取任务窗格实例
var taskPaneForm = AddinStudy.GetTaskPaneForm();

if (taskPaneForm != null)
{
    // 添加新的状态记录
    taskPaneForm.AddPartStatus(
        partName: "零件名称",
        status: "处理中",  // 或 "成功"、"失败" 等
        remark: "备注信息"
    );
    
    // 更新已有零件的状态
    taskPaneForm.UpdatePartStatus(
        partName: "零件名称",
        status: "已完成",
        remark: "更新备注"
    );
}
```

## 使用示例

### 示例1: 在批量处理中记录状态

```csharp
[Command(2001, "批量处理", "批量处理零件", "batch_process", (int)swDocumentTypes_e.swDocASSEMBLY)]
private void BatchProcess()
{
    var taskPaneForm = AddinStudy.GetTaskPaneForm();
    
    // 获取装配体中的所有零件
    // ... 遍历零件代码 ...
    
    foreach (var part in parts)
    {
        try
        {
            // 记录开始处理
            taskPaneForm?.AddPartStatus(part.Name, "处理中", "开始处理");
            
            // 执行处理逻辑
            ProcessPart(part);
            
            // 记录处理成功
            taskPaneForm?.UpdatePartStatus(part.Name, "成功", "处理完成");
        }
        catch (Exception ex)
        {
            // 记录处理失败
            taskPaneForm?.UpdatePartStatus(part.Name, "失败", ex.Message);
        }
    }
}
```

### 示例2: 检查K因子时记录状态

```csharp
[Command(1010, "检查K因子", "检查当前零件的K因子设置是否正确", "checkkfactor", (int)swDocumentTypes_e.swDocPART)]
private void CheckKFactor()
{
    var taskPaneForm = AddinStudy.GetTaskPaneForm();
    string partName = PartStatusForm.GetCurrentPartName(swApp);
    
    try
    {
        taskPaneForm?.AddPartStatus(partName, "处理中", "检查K因子");
        
        checkk_factor.run(swApp, swModel);
        
        taskPaneForm?.UpdatePartStatus(partName, "成功", "K因子检查通过");
    }
    catch (Exception ex)
    {
        taskPaneForm?.UpdatePartStatus(partName, "失败", ex.Message);
        throw;
    }
}
```

## 注意事项

1. **任务窗格自动初始化**: 插件加载时会自动创建任务窗格，无需手动操作
2. **窗口位置**: 任务窗格默认显示在SolidWorks右侧的任务窗格区域
3. **数据持久化**: 当前版本的状态数据仅保存在内存中，插件卸载后会清空
4. **线程安全**: 所有UI操作都在主线程执行，确保线程安全

## 扩展建议

如果需要更强大的功能，可以考虑：

1. **数据持久化**: 将状态保存到文件或数据库
2. **导出功能**: 导出状态记录为Excel或CSV
3. **过滤搜索**: 添加按名称、状态、时间过滤的功能
4. **统计信息**: 显示成功/失败/处理中的数量统计
5. **自动记录**: 在某些命令执行时自动记录状态，无需手动调用

## 故障排除

### 问题1: 任务窗格不显示
- 检查插件是否正确加载
- 尝试点击"显示零件状态窗格"命令
- 查看Debug输出是否有错误信息

### 问题2: 状态记录不成功
- 确认任务窗格已初始化
- 检查是否有打开的文档
- 查看Debug输出日志

### 问题3: 窗体显示异常
- 重启SolidWorks
- 重新加载插件
- 检查PartStatusForm.cs是否有编译错误
