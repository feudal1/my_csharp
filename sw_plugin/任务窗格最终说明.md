# 零件状态任务窗格 - 最终版说明

## ✅ 完成的功能

### 1. 自动刷新机制
- 打开装配体时自动调用asm2bom生成BOM
- BOM生成完成后自动提取数据并更新任务窗格
- 切换活动文档到装配体时也会触发刷新

### 2. 任务窗格显示内容
表格包含5列：
- **零件名称**: 点击可选中装配体中的对应零件
- **零件类型**: 钣金件/管件/其他
- **规格尺寸**: 零件的长x宽x高或下料尺寸
- **是否出图**: 已出图(绿色)/未出图(黄色)
- **数量**: 零件数量

### 3. 交互功能
- 点击零件名称 → 自动选中装配体中的零件并缩放视图
- 根据是否出图显示不同背景色
- 窗口大小可调整，最小400x400

## 📋 使用流程

### 方法1: 自动刷新（推荐）
1. 加载插件
2. 打开装配体文件
3. 等待几秒，BOM自动生成，任务窗格自动更新
4. 点击"显示零件状态窗格"查看结果

### 方法2: 手动触发
1. 打开装配体
2. 执行"装配体导出BOM"命令
3. BOM生成后任务窗格自动更新

## 🔧 技术实现

### 回调机制
```csharp
// asm2bom.cs中定义回调
public static Action<List<object>>? TaskPaneUpdateCallback { get; set; }

// addin.cs中设置回调
tools.asm2bom.TaskPaneUpdateCallback = OnBomDataUpdated;

// BOM生成完成后调用回调
UpdateTaskPaneFromBom(swTableAnnotation);
TaskPaneUpdateCallback?.Invoke(bomDataList);
```

### 事件监听
```csharp
swApp.FileOpenPostNotify      // 文件打开后触发
swApp.ActiveDocChangeNotify   // 活动文档变更时触发
```

### 数据流
```
打开装配体 
  ↓
触发FileOpenPostNotify事件
  ↓
调用RefreshTaskPaneForAssembly
  ↓
异步执行asm2bom.run()
  ↓
生成BOM表
  ↓
调用UpdateTaskPaneFromBom提取数据
  ↓
通过TaskPaneUpdateCallback回调
  ↓
OnBomDataUpdated处理数据
  ↓
taskPaneForm.LoadFromBomData更新界面
```

## ⚠️ 注意事项

1. **首次打开可能较慢**: 因为需要生成完整的BOM表
2. **大装配体**: 零件很多时可能需要等待较长时间
3. **Excel会弹出**: asm2bom会打开生成的Excel文件
4. **线程安全**: UI更新使用了Invoke确保在主线程执行

## 🎯 优化建议

如果不想每次都自动生成BOM，可以：
1. 注释掉addin.cs中的`RegisterDocumentEvents()`调用
2. 只在需要时手动执行"装配体导出BOM"命令
3. 或者添加一个"刷新任务窗格"按钮手动触发

## 🐛 故障排除

### 问题: 任务窗格没有内容
- 检查Console输出是否有"任务窗格已更新"的消息
- 确认BOM表成功生成（应该会弹出Excel）
- 检查是否有异常信息

### 问题: 点击零件名称没反应
- 确认当前是装配体文档
- 检查零件名称是否与装配体中的组件名称匹配
- 查看Debug输出是否有错误

### 问题: 打开装配体时卡住
- 这是因为在异步生成BOM
- 大装配体可能需要较长时间
- 可以考虑禁用自动刷新，改为手动触发
